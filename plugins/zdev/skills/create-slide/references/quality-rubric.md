# 品質ルーブリック（自己採点用）

スライド作成後に 9 次元で自己採点し、品質を担保する。

## 採点基準

9 次元 x 10 点 = **90 点満点**。**68 点以上**で合格。

| # | 次元 | 10点（優秀） | 7点（良好） | 4点（要改善） | 1点（不足） |
|---|------|------------|-----------|-------------|-----------|
| 1 | **学習ゴール明確性** | 冒頭で「何を学べるか」が具体的に明示されている | 学べることは書いてあるが抽象的 | 目次はあるが学習ゴールが不明確 | 学習ゴールの記載なし |
| 2 | **聴衆適合性** | 難易度・用語・例が対象者に一致している | 一部だけ難易度がズレる | 多くの説明が対象者に合っていない | 対象者が不明 |
| 3 | **構成の論理性** | 概念が積み上げ順に並び、前のスライドが次の前提になっている | 概ね論理的だが一部飛躍がある | 順番に違和感がある箇所が複数 | 順序がバラバラ |
| 4 | **見出しの説明力** | 見出しだけ読めば内容がわかる（説明的見出し） | 大部分は説明的だが一部が曖昧 | 「概要」「詳細」など汎用的な見出しが多い | 見出しが内容と無関係 |
| 5 | **認知負荷の適正** | 1 スライド 1 メッセージを守り、情報量が過不足ない | 一部で情報が多いが理解は可能 | 詰め込みが目立ち追いづらい | ほぼ全体が過密/過疎 |
| 6 | **視覚要素の活用** | コード/図/テーブルが各スライドに効果的に配置 | 一部テキストのみのスライドがある | 視覚要素が少なく文字中心 | 視覚要素がほぼない |
| 7 | **一貫性** | テーマ・用語・記法・コードスタイルが全体で統一 | 軽微な不統一が 1-2 箇所 | 不統一が目立つ | 統一感がない |
| 8 | **完全性** | 冒頭の学習ゴールが全てカバーされている | 1 つの学習ゴールが薄い | 複数の学習ゴールが未カバー | 大部分が未カバー |
| 9 | **事実正確性** | 全主張がソースコード・設定ファイルと一致し、コード例の API パターン（クラス名・メソッド名・引数）が実装と同一で、テーブルに列挙したクラス・コンポーネントの実在と役割が正確 | 1-2 箇所の軽微な数値ズレ | コード例の API が実装と不一致、テーブルに誤った役割や存在しないクラスを記載、自動/手動の混同 | 削除済み機能・存在しない設定を記載、コード例が実装と全く異なる |

## 採点ワークフロー

1. スライド全体を通して各次元を採点する
2. 合計が **68 点未満** → 下位 2 項目を特定し修正する（最大 2 回）
3. 合計が **68 点以上** → レビューループへ進む

## 完了報告への記載形式

```
### 自己採点: XX / 90

| 次元 | スコア |
|------|--------|
| 学習ゴール明確性 | X/10 |
| 聴衆適合性 | X/10 |
| 構成の論理性 | X/10 |
| 見出しの説明力 | X/10 |
| 認知負荷の適正 | X/10 |
| 視覚要素の活用 | X/10 |
| 一貫性 | X/10 |
| 完全性 | X/10 |
| 事実正確性 | X/10 |
```
