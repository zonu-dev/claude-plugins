# レビューチェック観点

PNG 画像を Read ツールで **8 枚ずつ並列** で読み取り、以下の観点でチェックする。

## チェック観点一覧

| # | 観点 | NG の判定基準 |
|---|------|------------|
| 1 | **下部はみ出し** | テキスト・テーブル・コードブロックがページ番号と**重なっている・接触している**場合は NG。余白が狭くても重なりがなければ OK |
| 2 | **右端はみ出し** | テキストやコードがスライド右端で切れている |
| 3 | **テーブル崩れ** | セルの内容が切れている、列が画面外に出ている |
| 4 | **コードブロック切れ** | コードの末尾が見切れている |
| 5 | **要素の重なり** | ヘッダー・フッター・ページ番号とコンテンツが重なっている。`header:` テキストと `##` 見出しの間に見出しフォント約 1 行分以上の余白がない場合も NG（コンテンツ過密による近接） |
| 6 | **空白過多** | コンテンツがスライドの上半分にしかなく、下半分が大きく空いている（`lead` クラスのタイトルスライドは除外） |
| 7 | **SVG 画像の問題** | 画像が表示されていない（空白）、画像がスライド外にはみ出している |
| 8 | **余白不足** | スライドの 40% 以上がコンテンツで占められ、余白が明らかに不足している |
| 9 | **コントラスト不足** | テキストと背景の区別がつきにくい箇所がある |
| 10 | **視覚要素の欠落** | タイトル・まとめ以外のスライドで、コード/図/テーブル/画像がないテキストのみのスライド |
| 11 | **フットノート重なり** | `<div class="footnotes">` のテキストがメインコンテンツと重なっている、またはページ番号で隠れている |
| 12 | **フットノート過多** | 1 スライドにフットノートが 4 件以上ある |
| 13 | **図中テキストの可読性** | Mermaid 図やチャート内のテキストが小さすぎて読めない、またはノードが密集して判読困難。ノード内テキストがスライド本文の半分以下のサイズに見える場合は NG |
| 14 | **矢印方向の整合性** | 依存関係図の矢印が「依存元 → 依存先」の向きでない、またはスライド本文中の `→` 表記と Mermaid 図の矢印方向が不一致 |
| 15 | **折り返し孤立文字** | `##` 見出しや `▶` 要点行が折り返して 2 行目に 1〜4 文字だけ残っている。[content-limits.md](../../create-slide/references/content-limits.md) の「折り返し孤立文字の防止」の判定上限を超えている場合も NG |
| 16 | **OSS ツールリンク欠落** | スライド中に名前が登場する OSS ツール・ライブラリ・GitHub Action に、初出ページでフットノートリンクがない。対象: ESLint, Prettier, Jest 等の有名ツールも含む全 OSS |
| 17 | **CJK 混合桁揃え計算誤り** | コードブロック内の CJK/ASCII 混在桁揃えで、表示幅（CJK=2, ASCII=1）ベースの計算に 1 スペース以上の誤りがある。CJK 文字数差によるフォント由来の微細なズレは許容する（**PNG 画像では検出不可、ソース分析でのみ検出**） |

## 判定ルール

- **実際に重なり・切れ・接触が確認できる場合のみ NG とする。** 余白が狭くてもコンテンツが収まっていれば OK
- [source-check-rules.md](source-check-rules.md) のソースコード分析で上限超過が検出されたページは、画像で問題が見えなければ**注意**として報告する（NG ではない）
- 図のテーマ切替チェックで NG があれば、レイアウトが問題なくても**要修正**として報告する
